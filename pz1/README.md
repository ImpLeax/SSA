### Завдання 1.2 

### Дослідіть, які бібліотеки доступні у вашій системі.

### 1. Використовуйте команду ldd, щоб перевірити, які бібліотеки використовуються вашими програмами (наприклад, ls або gcc).

### 2. Знайдіть бібліотеку з math-функціями, визначте її точне місце розташування (find /usr/lib -name "*math*").

### 3. Перевірте символи бібліотеки за допомогою nm або objdump. 

**Наприклад:**
```bash
 nm -D /usr/lib/libm.so | grep erf
```
### 4. Проаналізуйте залежності бібліотек за допомогою ldd або objdump -p.

### 5. Напишіть скрипт на Bash, який аналізує всі бібліотеки у /usr/lib, витягує з них символи та шукає певні функції (наприклад, sin, cos, exp).

### 1. Аналіз динамічних залежностей
За допомогою команди `ldd` було перевірено, які бібліотеки використовуються системними програмами.

**Приклад для `ls`:**
```bash
ldd /bin/ls
```

### 2. Пошук математичної бібліотеки
Для визначення точного місця розташування бібліотеки з math-функціями була використана команда `find`.
```bash
find /lib /usr/lib -name "*math*" 2>/dev/null
```

### 3. Перевірка символів бібліотеки
Використано утиліту `nm` для пошуку функції `erf` у знайденій бібліотеці.
```bash
nm -D /usr/lib/x86_64-linux-gnu/libm.so.6 | grep erf
```
<img width="728" height="327" alt="image" src="https://github.com/user-attachments/assets/e3f0706b-2516-4865-8937-003d3b2284ac" />

### 4. Поглиблений аналіз залежностей
Використано objdump -p для аналізу заголовків та перегляду списку необхідних бібліотек (`NEEDED`).
```bash
objdump -p /bin/ls | grep NEEDED
```
